- **2026-06-16 — Haseeb + Lovable** — **Feature 1B — Overpass calibration-gate unblock.** Live engine was failing every calibration run with `Accessibility lookup failed (overpass_highway_node, overpass_major_road_node)` because both public Overpass mirrors were returning `406` (request format) and `429` (rate limit). Root cause: edge function was POSTing the query as raw `text/plain` with no User-Agent, which both `overpass-api.de` and `overpass.kumi.systems` reject. Fix in `supabase/functions/_shared/mapbox.ts`: (1) switched body to `application/x-www-form-urlencoded` with `data=<urlencoded query>`, (2) added a real `User-Agent: neuron-garage-sas/0.2` + `Accept: application/json`, (3) added `https://maps.mail.ru/osm/tools/overpass/api/interpreter` as the primary mirror (verified reachable + returning real OSM nodes), kept overpass-api.de and kumi as additional fallbacks, (4) added a debug log line with node count + best distance per call. Also removed the silent `haversine × 1.3` fallback inside `drivingDistanceMiles` — Mapbox Directions failures now propagate as `null`, surfacing an explicit engine error per the "no synthetic numbers" rule. No UI / scoring-math change. Verified end-to-end: `compute-sas` for Trinity Christian Academy now returns `status=ready` with `highwayDistanceMi: 8.37`, `roadDistanceMi: 0.9` and pillar accessibility 33 / composite SAS 50.33 — all from live OSM + Mapbox data.
