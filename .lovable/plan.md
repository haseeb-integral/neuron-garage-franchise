# Wire "Export Comparison" button

Replace the current toast stub on the Compare modal's Export button with two real actions: **Export as Excel** and **Export as PDF**. Frontend-only ‚Äî no backend, no scoring math changes.

## What the user sees

The single "Export Comparison" button becomes a dropdown with two options:

- **Excel (.xlsx)** ‚Äî primary, multi-sheet workbook
- **PDF (.pdf)** ‚Äî 1-page snapshot of the modal

Filename pattern: `compare-nashville-louisville-2026-05-26.xlsx` / `.pdf` (lowercased city slugs joined by `-`, then ISO date).

## Excel file ‚Äî 3 sheets

**Sheet 1: Overview**
| Metric | Nashville, TN | Louisville, KY |
|---|---|---|
| Overall Score | 99 | 99 |
| Tier | A | A |
| Demand | 83 | 72 |
| TAM Teachers | 73 | 78 |
| Competitive Opportunity | 68 | 67 |

Pillar numbers come from the same `buildRecomputedPillarScores ‚Üí buildPillarView` pipeline the modal already uses, so the file matches the screen exactly.

**Sheet 2: Key Market Signals**
One row per signal (all 13 currently shown), one column per market. Values formatted the same way as the modal (`formatMetric`).

**Sheet 3: Weights Snapshot**
- Applied preset name (if any)
- Per-category master weight %
- Per-category sub-metric weights %
- Exported-at timestamp

This sheet is critical ‚Äî Compare numbers shift if sub-weights change later, so the file needs to record the recipe that produced today's numbers.

## PDF file ‚Äî 1 page

A clean letter-size PDF that mirrors the modal:
- Header: "Market Comparison" + date + applied preset name
- Table: same three sections (Overview / Category Scores / Key Market Signals)
- Footer: "Generated by Neuron Garage Franchise Intelligence ‚Äî {timestamp}"

Rendered via `jspdf` + `jspdf-autotable` (small libs, no headless-browser dependency). Not a screenshot ‚Äî a real text-based PDF so values are selectable/copyable.

## Technical notes

- Pillar values in the Excel and PDF use `buildRecomputedPillarScores(market, appliedSubWeights) ‚Üí buildPillarView` ‚Äî the exact same path the modal uses. No second source of truth.
- Composite score and tier use `buildMarketView(m).composite` + `tierFromDisplayScore` ‚Äî same as modal.
- Signal rows are built from the modal's existing `signalsByCity` state, passed into the builder so we don't re-fetch.
- Filename slug: `${city.toLowerCase().replace(/[^a-z0-9]+/g, '-')}` per market, joined with `-`, capped at 4 markets in the name (rest collapsed to `+N-more`).
- Both builders pure ‚Üí no `window`/`document` access ‚Üí easy to unit-test in vitest.
